Script location
---------------
scripts/qa/a2a_proxy_smoke.py — 13 scenario functions across two
groups (T9 card discovery, T10 dispatch + RBAC), driven by a main()
that prints a tabular pass/fail summary.

Scenarios covered
-----------------
T9 — Card discovery (4 scenarios):
  T9.1 card returns 200 + JSON
  T9.2 protocolBinding camelcase (NOT transportProtocol)
  T9.3 protocolBinding value == JSONRPC
  T9.4 URL rewritten to gateway-public (NOT upstream endpoint_url)

T10 — Dispatch envelope, methods, errors, version, aliases (7 scenarios):
  T10.1 body=[] -> -32600 (envelope validation)
  T10.2 malformed JSON -> -32700 (parse error)
  T10.3 A2A-Version=99.0.0 -> -32009 (unsupported version)
  T10.4 SendMessage -> result envelope (unary dispatch)
  T10.5 message/send v0.3 alias mapped (T7 + T4 alias handling)
  T10.6 tasks/list NOT a legacy alias -> -32601 (Oracle v3 #22)
  T10.7 SendStreamingMessage -> text/event-stream + parseable chunk

T10 — RBAC denial (2 scenarios):
  T10.8 missing Authorization -> 401 (transport-level)
  T10.9 invalid token -> 401

Prerequisites
-------------
1. Live ContextForge gateway at A2A_COMPLIANCE_GATEWAY_URL (default:
   http://localhost:4444).
2. Bundled a2a_echo_agent at A2A_ECHO_BASE_URL (default:
   http://127.0.0.1:9100).
3. Echo agent registered with the gateway as
   A2A_COMPLIANCE_AGENT_NAME (default: a2a-echo-agent).
4. `make testing-up` typically satisfies all three (compose 'testing'
   profile brings up gateway + nginx + echo agent on port 9100).
5. Bearer token in MCPGATEWAY_BEARER_TOKEN env. If unset, falls back
   to tests.helpers.auth.make_test_jwt (which signs with the same
   JWT_SECRET_KEY the gateway uses) for dev-mode runs.

Usage
-----
$ python scripts/qa/a2a_proxy_smoke.py

Or with overrides:
$ A2A_COMPLIANCE_GATEWAY_URL=http://localhost:4444 \\
  A2A_COMPLIANCE_AGENT_NAME=a2a-echo-agent \\
  MCPGATEWAY_BEARER_TOKEN=eyJ... \\
  python scripts/qa/a2a_proxy_smoke.py
