Acceptance criteria (per plan)
------------------------------
- Route: GET /a2a/{agent_name}/.well-known/agent-card.json
- Public (no @require_permission, no auth Depends)
- Calls synthesize_agent_card with user_email=None, token_teams=[]
- AgentCard return → 200 + JSON with by_alias=True + exclude_none=True
- None return → 404
- a2a_service is None → 503
- Reads server_id from request.scope["a2a_server_id"] (Wave 4 W16 middleware populates this)
- Same handler serves v-server URL post-middleware-rewrite

Handler location
----------------
mcpgateway/main.py: get_a2a_agent_card() — inserted between
existing @a2a_router.post("/{agent_name}/invoke") (line 5041)
and @a2a_router.post("/invoke") (line 5144) so per-agent routes
stay grouped together.

Notes
-----
- response_model is deliberately NOT used on the route decorator
  because FastAPI's response_model serializes by Python attribute
  name, not by alias — that would silently corrupt the wire shape
  (Plan T11 Must-NOT-do #1).
- token_teams=[] is explicit (NOT None). Plan T11 Must-NOT-do #2
  guards against admin-bypass leaks via _check_agent_access:
  permission_service.py:126-130 suppresses admin bypass when
  token_teams=[].
- public_base resolution uses getattr() because a2a_public_base_url
  is a SOFT setting that may or may not be defined; app_domain is
  the always-present fallback (Oracle re-review #4).
