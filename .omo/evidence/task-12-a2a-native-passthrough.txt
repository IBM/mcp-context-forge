QA scenario mapping (per plan T12)
----------------------------------
(a) malformed JSON → -32700
    → test_malformed_json_returns_32700

(b) body=[] → -32600
    → test_non_dict_body_returns_32600

(c) missing agent → 404
    → test_agent_not_found_returns_404 (mocks resolve to raise
       A2AAgentNotFoundError, which is the same exception T3 raises for
       wrong-team / wrong-server visibility miss per D14)

(d) wrong-team token (token_teams=[]) for team-only agent → 404
    → SAME wire outcome as (c): A2AAgentNotFoundError. Covered by
       test_agent_not_found_returns_404 + permission_service.py:126-130
       admin-bypass-suppression contract verified by Wave 1 T3 unit tests.

(e) A2A-Version: 99.0.0 → -32009
    → test_unsupported_version_returns_32009

(f) GetExtendedAgentCard with a2a.read only (NO a2a.invoke) → 200 + card
    → test_extended_card_returns_synthesized_card. Permission grant True;
       synthesizer mocked to return the card. Crucial assertion:
       mock_dispatch_service.dispatch_a2a_jsonrpc_unary.assert_not_called()
       proves the GetExtendedAgentCard branch short-circuits at step 8
       (D18 NEVER forward upstream).

(g) SendMessage with a2a.invoke only → 200 + result
    → test_invoke_success_returns_result_envelope

(h) SendMessage with a2a.read only → 403
    → test_invoke_without_permission_returns_403. The handler calls
       check_permission(permission='a2a.invoke', ...) for non-extended
       methods; granting only a2a.read returns False here.

(i) public-only token (token_teams=[]) for ADMIN → 403 (admin bypass
    suppressed at permission_service.py:126-130)
    → CONTRACT: the dispatch_overrides fixture seeds
       filter_context=(..., [], False) — non-admin public-only. The
       handler then calls check_permission(token_teams=[]); the
       permission_service module's admin-bypass-suppression logic
       (lines 126-130 — when token_teams==[], admin bypass is
       suppressed) is the contract. Wave 1 T3 unit tests
       (tests/unit/mcpgateway/services/test_a2a_service_native.py)
       verify the supression mechanism directly. The integration
       test test_invoke_without_permission_returns_403 covers the
       wire outcome.

Notes
-----
- CSRF middleware: dispatch tests send Authorization: Bearer fake-token
  to bypass CSRF (csrf_middleware.py:113-116 skips Bearer-authenticated
  requests). The token value is irrelevant because dependency overrides
  intercept auth before token validation. Real Bearer tokens are
  produced by tests/helpers/auth.make_test_jwt for live-gateway tests.
- Cache-Control assertion accepts both "no-cache" (handler's value) and
  "no-store" (security middleware's stricter override). Both satisfy
  the "do not cache" contract for streaming responses.
- The dispatch_overrides fixture patches get_rpc_filter_context at the
  module-attribute level rather than going through real JWT verification
  because TestClient does not populate request.state.bearer_token /
  request.state._jwt_verified_payload (those are set by the auth
  middleware in production).
- Route ordering: /a2a/{agent_name} POST is registered AFTER
  /a2a/{agent_name}/invoke POST so the longer path stays unambiguous.
  T13 (route-ordering regression test) explicitly asserts this in
  Wave 3's next todo.
