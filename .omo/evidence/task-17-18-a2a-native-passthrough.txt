Wave 4 Phase A composition note
-------------------------------
T17 + T18 verify the FULL Phase A stack end-to-end:

   middleware (T16) → handler (T11 / T12) → service
   → a2a_access_policy (Phase A centralized RBAC/ABAC) → primitives

Per the user-clarified three-level conjunctive model:

   GET /servers/{id}/a2a/{name}/.well-known/agent-card.json
   denies unless ALL THREE pass:
     1. Server visibility (cheap, sync, no DB)
     2. Agent-in-server membership (single SELECT)
     3. Agent-level visibility (async, may query teams)

   POST /servers/{id}/a2a/{name}
   same three-level denial wraps the dispatch + permission checks
   from T12 (a2a.invoke / a2a.read per method).

T17 — TestVirtualServerCardEndpoint
------------------------------------
1. test_happy_path_serves_card_and_passes_server_id
   GET /servers/srv-abc/a2a/echo/.well-known/agent-card.json
   → 200 + JSON card. Asserts the synthesize_agent_card mock was
   called with server_id="srv-abc" (proves T16 middleware ran AND
   the T11 handler propagated the scope key) AND with
   user_email=None / token_teams=[] (proves the D11 public-discovery
   contract still holds on the v-server URL).

2. test_denial_returns_404
   Synthesizer returns None (any denial reason). Response: 404.
   Asserts server_id was still passed (the middleware ran even
   though the denial short-circuited downstream).

3. test_no_authorization_required
   GET v-server URL without auth headers → 200. Confirms the
   v-server card endpoint stays PUBLIC (D11) just like the per-agent
   URL.

T18 — TestVirtualServerDispatchEndpoint
----------------------------------------
1. test_happy_send_message_resolves_with_server_id
   POST /servers/srv-x/a2a/echo with SendMessage payload → 200 +
   JSON-RPC result envelope {"jsonrpc": "2.0", "result": {"task":
   "done"}, "id": "abc"}. Asserts resolve_agent_for_dispatch was
   called with server_id="srv-x".

2. test_agent_not_in_server_returns_404
   resolve_agent_for_dispatch raises AgentNotInServerError →
   HTTP 404. This is the test that originally failed and drove the
   handler exception-handling fix. All four denial reasons now
   collapse to 404.

3. test_streaming_via_vserver_url_returns_sse
   POST /servers/srv-y/a2a/echo with SendStreamingMessage →
   200 + Content-Type: text/event-stream. Mock streaming generator
   yields a chunk; the T14 _sse_format wraps it as
   "data: {json}\n\n". Asserts the resolver received server_id="srv-y"
   on the streaming code path too.

4. test_extended_card_via_vserver_respects_a2a_read
   POST /servers/srv-z/a2a/echo with GetExtendedAgentCard +
   permission_grant=True → 200 + JSON-RPC result carrying the card.
   Crucial: assert_not_called() on dispatch_a2a_jsonrpc_unary
   confirms the D18 invariant (GetExtendedAgentCard NEVER forwards
   upstream) still holds via the v-server URL. Asserts synthesizer
   received server_id="srv-z".
